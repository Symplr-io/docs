Create applications using prebuilt templates that include reusable screens, workflows, layouts, and navigation structures.

Templates provide reusable application foundations that can be customized and extended within Symplr.

## Why It Matters

Using templates helps teams:

- Start with prebuilt application structures and workflows
- Reduce manual setup and configuration effort
- Support faster prototyping and development
- Reuse common business application patterns
- Customize generated applications based on project requirements

## Overview

Symplr allows applications to be generated from reusable templates directly from the application creation workspace.

Templates include predefined screens, navigation flows, layouts, and configuration settings that can be customized within Blueprint View.

## What You Can Do with Templates

Templates allow you to:

- Create applications using reusable predefined structures
- Use prebuilt screens, layouts, and navigation flows
- Configure application settings before generation
- Customize generated applications within Blueprint View
- Preview and validate applications using Build & Run workflows

## Quick Start

The following workflow summarizes the template-based application creation process:

1. Open the Templates gallery
2. Browse and select a template
3. Configure application settings
4. Generate the application
5. Review the generated Blueprint structure
6. Customize screens and workflows
7. Build and preview the application

## Before You Begin

Before creating applications using templates, ensure:

- You have access to the Symplr workspace
- Template features are available
- The application requirements are identified
- You are ready to configure application settings

## Open the Templates Gallery

From the application workspace, click **Create from Template**.

<Frame>
  ![1 Templates Entry Point](/images/1-Templates-Entry-Point.jpg)
</Frame>

Symplr opens the Templates gallery containing reusable application templates.

<Frame>
  ![2 Templates Gallery](/images/2-Templates-Gallery.jpg)
</Frame>

The Templates gallery allows templates to be browsed and selected for application generation.

## Select a Template

Browse the available templates and select the application structure that matches your requirements.

<Frame>
  ![3 Selecting Template](/images/3-Selecting-Template.jpg)
</Frame>

Templates may include predefined screens, navigation structures, workflows, and reusable layouts.

## Configure the Application

After selecting a template, configure the required application settings.

<Frame>
  ![4 Template Configuration](/images/4-Template-Configuration.jpg)
</Frame>

Configuration settings may include:

- Application name
- Application theme
- React Native version
- Package name

After configuring the required settings, click **Continue** to begin generating the application.

## Generate the Application

Symplr begins generating the application structure using the selected template and configuration settings.

<Frame>
  ![5 Template Generation](/images/5-Template-Generation.jpg)
</Frame>

The generation process prepares application screens, navigation, layouts, and workflows automatically.

## Review the Generated Blueprint

Once generation completes successfully, the generated application becomes available within Blueprint View.

<Frame>
  ![6 Generated Blueprint](/images/6-Generated-Blueprint.jpg)
</Frame>

Blueprint View displays the generated application structure, including screens, workflows, and navigation structures.

## Review and Customize Generated Screens

Generated screens can be reviewed and customized within the screen editor.

<Frame>
  ![7 Review Generated Screens](/images/7-Review-Generated-Screens.jpg)
</Frame>

You can continue configuring widgets, layouts, workflows, navigation settings, and themes based on application requirements.

## Build and Preview the Application

After reviewing the generated application, use Build & Run workflows to preview and validate the application directly within Symplr.

<Frame>
  ![8 Build Run Template App](/images/8-Build-Run-Template-App.jpg)
</Frame>

Build & Run workflows help validate layouts, navigation behavior, screen rendering, and overall application behavior before deployment.

## Expected Outcome

After completing the template generation workflow, you will have:

- A generated application structure
- Prebuilt screens and navigation flows
- Editable workflows and layouts
- A customizable Blueprint View project
- An application ready for preview, testing, and continued development

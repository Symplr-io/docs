---
title: "Download Application Definition"
---

Download the Application Definition file to access the complete structural representation of your application in JSON format.

The Application Definition contains the underlying configuration and structure used by Symplr to represent your application, including screens, navigation, workflows, widgets, and related application settings.

## Before You Begin

Before downloading the Application Definition file, ensure that:

- Your project has been created or opened successfully
- You have access to the **Blueprint View**
- The application structure is available within the project

### Step 1: Open the Blueprint View

Open your project and navigate to the **Blueprint View**. The Blueprint View provides a visual representation of your application's screens, navigation, workflows, and overall structure.

From this view, you can access the option to download the complete Application Definition file.

### Step 2: Download the Application Definition

From the top-right corner of the Blueprint View, click the **“Download App Definition”** icon.

<Frame>
  ![Image](/images/image-57.png)
</Frame>

Symplr generates and downloads the Application Definition file to your local machine in JSON format.

Once the download completes successfully, Symplr displays a confirmation message.

<Frame>
  ![Image](/images/image-58.png)
</Frame>

### Step 3: Access the Downloaded File

After the download is complete, the Application Definition file becomes available within your local downloads directory.

<Frame>
  ![Image](/images/image-60.png)
</Frame>

The downloaded file is provided in `.json` format and typically includes the application name within the file name.

## What the Application Definition File Contains

The downloaded Application Definition file contains the complete structure and configuration of your application in JSON format.

This includes information such as:

- Application screens and navigation
- Widget structures and widget properties
- Workflows and application logic
- Application-level configuration and settings

The Application Definition acts as the underlying representation of your application and is used internally during application generation and code generation processes.

## Next Steps

After downloading the Application Definition file, you can continue with:

- Downloading the generated application code
- Running the application locally
- Configuring integrations and authentication
- Extending the application with additional screens and workflows

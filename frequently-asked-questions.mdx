The FAQ section provides answers to commonly encountered questions and issues when using Symplr.

Use this page to quickly troubleshoot problems and find guidance before contacting support.

## Quick Start

<AccordionGroup>
  <Accordion title="What is Symplr used for?">
    Symplr is a no-code platform that allows you to generate **mobile** and **Web** applications without development experience. You can create applications by importing a Figma file or by using AI prompts.
  </Accordion>

  <Accordion title="Do I need coding experience to use Symplr?">
    No. Symplr is designed for users without coding experience. The platform automatically generates application screens, navigation, and structure based on your input.
  </Accordion>

  <Accordion title="What type of applications can I build with Symplr?">
    Symplr allows you to generate mobile and web applications based on UI designs or AI prompts, including multi-screen apps with navigation and interactive components.
  </Accordion>

  <Accordion title="Is there a limit to the number of screens I can generate?">
    No. The number of screens depends on the complexity of your design or prompt. Larger designs may take longer to process.
  </Accordion>

  <Accordion title="How can I find my existing projects in Symplr?">
    You can view all your projects in the **Home tab** under the **“Your App”** section. You can also search for a specific project by entering the project name in the search box.
  </Accordion>

  <Accordion title="Can I search for a project in Symplr?">
    Yes. You can quickly find a project by typing its name in the **search bar** within the Home tab.
  </Accordion>

  <Accordion title="Can I delete a project?">
    Yes. You can delete a project directly from the **“Your App”** section in the Home tab.
  </Accordion>
</AccordionGroup>

## Figma Integration or AI Generation

<AccordionGroup>
  <Accordion title="How can I generate an application in Symplr?">
    You can generate an application in two ways:

    - Importing a **Figma File URL**
    - Using **AI prompts** to describe your application

    Both methods generate a complete application structure.
  </Accordion>

  <Accordion title="What happens after I import a Figma file or use AI prompts?">
    Symplr generates a **Storyboard View (Blueprint View)** that displays all screens, screen names, and navigation flow of the application.
  </Accordion>

  <Accordion title="Can I update my design after importing it?">
    Yes. You can modify your Figma file and re-import it to regenerate an updated **Storyboard View**.
  </Accordion>

  <Accordion title="Which method should I use — Figma import or AI prompt?">
    Choose one based on your requirement:

    - Use **Figma import** if you already have a design
    - Use **AI prompts** if you want to generate an app without a design
  </Accordion>

  <Accordion title="Can I edit the application after generating it using AI?">
    Yes. You can review the generated screens in the **Storyboard View** and make updates by refining your prompt or re-generating the application.
  </Accordion>

  <Accordion title="What happens if my Figma design is complex?">
    Complex designs may take longer to process, and some components may require adjustments to ensure accurate screen generation.
  </Accordion>
</AccordionGroup>

## View Storyboard & Screens

<AccordionGroup>
  <Accordion title="What is the Storyboard View?">
    The **Storyboard View (Blueprint View)** is a visual representation of your application, showing all screens and their navigation connections.
  </Accordion>

  <Accordion title="Can I view individual screens in detail?">
    Yes. You can navigate to the **“Pages” tab** and select screens from the **“Screens” panel** to view each screen individually.
  </Accordion>

  <Accordion title="Can I edit screens directly in Symplr?">
    Yes. You can open individual screens in Symplr and make adjustments directly within the platform.

    For more significant design changes, you can update your Figma file or AI prompt and regenerate the application to reflect those updates.
  </Accordion>

  <Accordion title="How is navigation between screens created?">
    Navigation is automatically generated based on the structure and relationships defined in your Figma design or AI prompt.

    You can also manually add or update navigation between screens directly from the **Storyboard View**, giving you full control over the application flow.
  </Accordion>

  <Accordion title="What should I do if navigation is incorrect?">
    If the navigation is incorrect, you can update it directly in the **Storyboard View** by modifying or adding navigation between screens.

    If the issue originates from the design or structure, you can update your Figma file or AI prompt and regenerate the application to apply the changes.
  </Accordion>
</AccordionGroup>

## Build & Run App

<AccordionGroup>
  <Accordion title="Can I build and run the application within Symplr?">
    Yes. You can build and run the application directly within Symplr from the **“Build & Run App”** section.
  </Accordion>

  <Accordion title="Can I run the application on my mobile device?">
    Yes. You can run the application on a local device by scanning a QR code using the **Expo Go** application.
  </Accordion>

  <Accordion title="What happens if the build fails?">
    An error message will be displayed. The build failures may occur due to configuration issues or unsupported elements. Review your setup and try rebuilding the application.
  </Accordion>

  <Accordion title="Can I rebuild the application after making changes?">
    Yes. You can rebuild the application after updating your design or configuration.
  </Accordion>
</AccordionGroup>

## Download Code & Test

<AccordionGroup>
  <Accordion title="Can I download the generated code?">
    Yes. Symplr allows you to download the generated application code, which can be run on an emulator.
  </Accordion>

  <Accordion title="Does Symplr support automated testing?">
    Yes. Symplr can generate automated test cases for your application, allowing you to validate functionality and ensure the application works as expected.
  </Accordion>

  <Accordion title="How can I test my application?">
    You can test your application by:

    - Running it within Symplr
    - Running it on a mobile device using Expo Go
    - Running the downloaded code in an emulator and executing automated test cases
  </Accordion>

  <Accordion title="Can I modify the downloaded code?">
    Yes. The generated code can be edited and customized in your local development environment.
  </Accordion>

  <Accordion title="What is included in the downloaded code package?">
    The package includes application code and generated test cases.
  </Accordion>

  <Accordion title="Can I run the application without downloading the code?">
    Yes. You can run the application directly within Symplr or on a local device using Expo Go.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Why are some screens missing after import?">
    This may happen if the Figma design is not properly structured (e.g., missing frames or inconsistent layout).
  </Accordion>

  <Accordion title="Why is my application not running on my device?">
    This may happen if the steps were not followed correctly.

    Ensure that:

    - Your device and system are connected to the same network
    - Expo Go is installed and updated
    - The QR code is scanned correctly
  </Accordion>

  <Accordion title="Why does my design look different in Symplr?">
    Differences may occur due to layout inconsistencies or unsupported design elements in Figma.
  </Accordion>

  <Accordion title="Why are some components not generated correctly?">
    This may happen if the components are deeply nested or not structured properly in Figma.
  </Accordion>

  <Accordion title="Why is my QR code not working?">
    Ensure the following:

    - Expo Go is installed
    - The QR code has not expired
  </Accordion>
</AccordionGroup>

<AccordionGroup>

    Yes. Symplr allows you to configure APIs and workflows at the screen level, giving you flexibility to manage functionality for each screen independently.
  </Accordion>
</AccordionGroup>

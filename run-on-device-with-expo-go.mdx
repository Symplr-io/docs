---
title: "Run on Device with Expo Go"
---

Run your application directly on a physical mobile device using Expo Go to validate layouts, navigation, responsiveness, and overall user experience in real time.

This allows you to experience and validate the application exactly as an end user would—without downloading or configuring the source code locally.

## Before You Begin

Before running your application on a device, ensure that:

- Your project has been created or opened successfully
- The application has been built successfully from the Blueprint View
- Your mobile device has internet access
- Expo Go is installed on your mobile device

### Step 1: Build the Application

Open your project and navigate to the **Blueprint View**. Before the application can run on a physical device, it must first be built successfully within Symplr.

Building the application prepares the latest version of the application for testing and generates the live preview required for device access. Once the build process completes successfully, the application preview becomes available within Symplr.

### Step 2: Run the Application on Your Device

From the application preview screen, click the **“Run App on Your Device”** button located in the top-right corner.

<Frame>
  ![Image](/images/image-44.png)
</Frame>

Symplr then generates a secure QR code that allows the latest version of the application to run directly on your mobile device using Expo Go.

### Step 3: Install Expo Go

After clicking the button, Symplr displays the generated QR code along with instructions for connecting your mobile device.

<Frame>
  ![Image](/images/image-45.png)
</Frame>

Expo Go is a mobile application that allows you to instantly run and preview your generated application on a physical device without requiring local development setup.

If Expo Go is not already installed on your mobile device, install it using the appropriate platform store:

- On iOS, install Expo Go from the App Store
- On Android, install Expo Go from the Google Play Store

The setup instructions are also displayed directly within Symplr.

<Frame>
  ![Image](/images/image-46.png)
</Frame>

<Note>
  Keep this screen open while running the application on your device. Closing the screen may stop the application from running.
</Note>

### Step 4: Scan the QR Code

Open Expo Go on your mobile device and scan the generated QR code to connect the application to your device.

#### iOS Devices

On iOS devices:

- Open the Camera or Code Scanner
- Scan the QR code displayed in Symplr
- Tap the prompt to open the application in Expo Go

<Frame>
  ![Image](/images/image-50.png)
</Frame>

#### Android Devices

On Android devices:

- Open the Expo Go application
- Use the built-in QR scanner
- Scan the QR code displayed in Symplr

<Frame>
  ![Image](/images/image-49.png)
</Frame>

Within seconds, the application loads directly on your mobile device and becomes ready for testing.

- Review screens and layouts for responsiveness and visual accuracy
- Interact with buttons and controls to confirm functionality
- Navigate between screens to validate user flows
- Ensure the overall experience is smooth, intuitive, and aligned with your design

## Troubleshooting

### QR Code Does Not Scan

- Ensure your mobile device camera permissions are enabled
- Make sure the QR code is fully visible on screen
- Verify that your mobile device has internet connectivity

### Application Stops Running

If the application stops unexpectedly:

- Ensure the Symplr preview screen remains open
- Rebuild the application if required
- Re-scan the QR code from Expo Go

---
title: "Manage Projects"
---

Symplr allows projects to be created, accessed, searched, organized, and removed directly from the Projects dashboard.

The Projects dashboard provides centralized access to application projects created within the platform, including projects generated from AI prompts, imported Figma designs, and template-based application structures.

## Before You Begin

Before managing projects within Symplr, ensure that:

- You have access to the Symplr workspace
- Existing projects are available within the dashboard
- The required project permissions are available
- The application workspace has loaded successfully

## Access the Projects Dashboard

After signing in to Symplr, the Projects dashboard displays all available application projects within your workspace.

Projects are displayed as cards containing the project name together with the most recent update information.

<Frame>
  ![1 Projects dashboard](/images/1-Projects-dashboard.jpg)
</Frame>

The dashboard also provides options for project creation, search, sorting, opening existing projects, and project removal workflows.

## Create a New Project

Symplr supports multiple project creation workflows depending on how the application development process should begin.

### Create a Project from Figma

To generate a project using an existing Figma design, select the **Import from Figma** option from the project creation section.

<Frame>
  ![2 Import From Figma](/images/2-Import-From-Figma.jpg)
</Frame>

This workflow allows application structures and screens to be generated from imported Figma designs.

### Create a Project from a Template

To begin using a predefined application structure, select the **Create from Template** option.

<Frame>
  ![3 Create From Template](/images/3-Create-From-Template.jpg)
</Frame>

Templates provide preconfigured application structures that can be extended and customized after project creation.

### Create a Project Using an AI Prompt

To generate a project using an application idea or prompt, enter the application description and click **Start Building**.

<Frame>
  ![4 Start Building from AI Prompt](/images/4-Start-Building-from-AI-Prompt.jpg)
</Frame>

Symplr then begins generating the initial application structure based on the provided prompt.

## View Existing Projects

All available projects are displayed within the dashboard as project cards.

Each project card includes:

- Project name
- Recent update information
- Project access controls
- Project management options

<Frame>
  ![5 Project cards](/images/5-Project-cards.jpg)
</Frame>

Projects remain accessible from the dashboard for continued development and management workflows.

## Search Projects

To locate a specific project, use the search field available above the project list.

<Frame>
  ![6 Search bar](/images/6-Search-bar.jpg)
</Frame>

The search workflow filters projects using matching project names and available project information.

## Sort Projects

Projects can be organized using the sorting dropdown available within the dashboard.

<Frame>
  ![7 Sort dropdown](/images/7-Sort-dropdown.jpg)
</Frame>

Symplr supports sorting projects using:

- Newest
- Oldest

This helps organize projects based on recent activity and update history.

## Open an Existing Project

To continue working on an existing project, select the project card and click the **Open** button.

<Frame>
  ![8 Open project button](/images/8-Open-project-button.jpg)
</Frame>

Opening a project loads the application directly into the **Blueprint View** for continued development and configuration workflows.

## Delete a Project

Project management options are available from the three-dot menu displayed on each project card.

<Frame>
  ![9 Three-dot menu](/images/9-Three-dot-menu.jpg)
</Frame>

To permanently remove a project, select the **Delete** option from the project menu.

Symplr then displays a confirmation dialog before removing the selected project.

<Frame>
  ![10 Delete confirmation dialog](/images/10-Delete-confirmation-dialog.jpg)
</Frame>

Project deletion is permanent and cannot be undone after confirmation.

## What You Can Manage from the Projects Dashboard

The Projects dashboard can be used to:

- Create new application projects
- Open existing projects
- Search available projects
- Sort projects by update history
- Access recent projects
- Remove unused projects
- Continue application development workflows

## Next Steps

After opening or creating a project, you can continue with:

- Configuring application structures
- Editing screens and workflows
- Building and previewing applications
- Downloading generated application code
- Managing authentication and database configurations

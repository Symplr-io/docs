## Overview

Bringing your Figma designs into Symplr Studio allows you to seamlessly transform static UI designs into a structured, interactive application blueprint. Instead of rebuilding screens manually, Symplr intelligently interprets your design and converts it into screens, components, and a foundation for your app’s logic and navigation.

This guide walks you through how to import your Figma designs and prepare them for further development within Symplr.

With Symplr, you can:

- Import a Figma design using a URL
- Automatically generate application screens and navigation
- Preview, build, and run the generated application

## Quick Start (2 Minutes)

1. Copy your **Figma File URL**
2. Import it into **Symplr**
3. Configure application settings
4. Review generated screens
5. Build and run your application

## Prerequisites

Before you begin, ensure you have the following:

- Access to a **Figma design file** with a valid shareable link
- A **Symplr account** with appropriate permissions
- Stable internet connectivity

## ⭐ **Preparing Your Figma File**

To achieve the best results, ensure your Figma file is properly structured:

- Use **frames** to represent screens
- Provide **clear and consistent screen names**
- Avoid deeply nested groups
- Use **reusable components** where possible
- Apply **Auto Layout** for responsive designs

<Note>
  **Important:**\
  Ensure your Figma file is accessible (e.g., _“Anyone with the link can view”_ or properly authorized).
</Note>

## Importing Figma Files into Symplr

Importing a Figma file is the first step in using Symplr. This process allows the platform to access your design assets and prepare them for code generation.

**Steps to Import Figma File**:

**Step 1: Copy the Figma File URL**

Open your Figma design and copy the shareable file URL.

<Frame>
  ![Image](/images/image-8.png)
</Frame>

**Step 2: Launch the Symplr Application**

Log in to your Symplr account and navigate to the **Home** page. Click on the **“Import from Figma”** button to begin the import process.

<Frame>
  ![Image](/images/image-9.png)
</Frame>

**Step 3: Provide the Figma URL**

Paste the copied Figma URL into the input field and click **“Continue”.** Ensure that the URL is valid and accessible.

<Frame>
  ![Image](/images/image-10.png)
</Frame>

<Note>
  **Note:**

  If you are connecting your Figma account to Symplr for the first time, you will be prompted to **authorize access**. Follow the on-screen instructions to complete the authorization process.
</Note>

**Step 4: Configure Application Settings**\
A configuration pop-up will appear. Provide the following details:

- **App Name** → Used to identify your project
- **Theme** → Defines default styling
- **React Native Version** → Ensures compatibility
- **Package Name** → Required for building and deployment

After entering all required details, click **"Continue".**

<Frame>
  ![Image](/images/image-12.png)
</Frame>

<Note>
  App Name must not contain spaces
</Note>

**Step 5: Initiate Import Process**

Symplr will begin importing the selected Figma file and processing its design elements. This may take a few moments depending on the size and complexity of the design.

<Frame>
  ![Image](/images/image-13.png)
</Frame>

**Step 6: Validate the Import**

After the import is complete, Symplr displays the **Storyboard View** showing all imported screens.

**Expected Result**

You should see:

- All screens listed in the storyboard
- No missing or broken components
- Proper navigation between screens

If issues are found:

- Update your Figma file
- Re-import into Symplr

<Frame>
  ![Image](/images/image-14.png)
</Frame>

## View each screen individually

After importing your Figma file and reviewing the Storyboard view, Symplr allows you to inspect each screen individually. This helps ensure that all design elements have been correctly interpreted before proceeding to code generation.

**Steps to View Individual Screens:**

**Step 1: Access the Storyboard View**

Once the import process is complete, Symplr will display the **Storyboard View** showing the overall structure of the application screens.

**Step 2: Navigate to the "Pages" Tab**

From the main interface, click the **“Pages”** tab. This section organizes the imported design into pages and associated screens.

<Frame>
  ![Image](/images/image-21.png)
</Frame>

**Step 3: View Default Screen**

Upon opening the Pages tab, the **first screen is displayed by default** in the main viewing area.

<Frame>
  ![Image](/images/image-22.png)
</Frame>

**Step 4: Browse Available Screens**

In the **“Screens” panel**, you will see a list of all available screens associated with the selected page.

<Frame>
  ![Image](/images/image-24.png)
</Frame>

**Step 5: Select and View Other Screens**

Click on any **screen name** within the Screens panel to load and view that specific screen in the main display area.

## Build & Run the Application

After generating and downloading the code, Symplr enables you to build and run your application both within the platform and on your own device, allowing you to validate functionality before deploying it.

### Build and Run the App in Symplr

Run your application directly within Symplr to quickly test functionality without setting up a local environment.

**Step 1: Navigate to Build & Run App dropdown**

From the **top-right corner** of the Symplr interface, click on **“Build & Run App.”** This section allows you to compile and execute your application.

**Step 2: Initiate the Build Process**

Click on the **“Build”** button to start compiling the application. The system will begin processing the generated code and preparing it for execution.

<Frame>
  ![Image](/images/image-17.png)
</Frame>

**Step 3: Monitor Build Progress**

Wait for the build process to complete. The system may display progress indicators during this step.

<Frame>
  ![Image](/images/image-18.png)
</Frame>

\
**Step 4: Run the Application**

Once the build is successful, it will proceed to run the application within the Symplr environment.

<Frame>
  ![Image](/images/image-25.png)
</Frame>

### Run Your Application Locally on Your Device

Run your application on your local device to test real-world behavior and validate performance before deployment.

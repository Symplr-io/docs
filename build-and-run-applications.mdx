---
title: "Build & Run Applications"
---

Symplr allows applications to be built and previewed directly within the platform before downloading or deploying the generated application package.

The Build & Run capability helps validate application screens, navigation flows, layouts, forms, interactions, and overall application behavior during runtime preview.

## Before You Begin

Before building the application, ensure your project contains configured screens, workflows, and navigation within the Blueprint View.

## Access the Build & Run Menu

Open your project in the **Blueprint View** and access the **Build & Run** dropdown from the top right corner of the workspace.

<Frame>
  ![1 Blueprint View Before Build](/images/1-Blueprint-View-before-build.jpg)
</Frame>

The **Build & Run** option provides access to application preview and runtime validation features.

<Frame>
  ![2 Build & Run Option](/images/2-Build-&-Run-option.jpg)
</Frame>

## Start a New Application Build

From the **Build & Run** menu, select **Build** under **Build & Run New App** to generate an interactive application preview.

<Frame>
  ![3 Build & Run button](/images/3-Build-&-Run-button.jpg)
</Frame>

Symplr begins generating the application preview using the current application structure and configuration.

The build duration may vary depending on the size and complexity of the application.

## Preview the Running Application

Once the build completes successfully, the application preview opens directly within the browser.

<Frame>
  ![6 Successful application preview running in browser](/images/6-Successful-application-preview-running-in-browser.jpg)
</Frame>

You can interact with the running application to validate layouts, navigation flows, forms, workflows, and overall user experience directly within the preview environment.

<Frame>
  ![7 Validation example](/images/7-Validation-example.jpg)
</Frame>

## Load a Previously Generated Build

If a previous application build already exists, Symplr allows the latest generated preview to be loaded directly without rebuilding the application.

From the **Build & Run** menu, click the **Load** button under **Load Latest App**.

<Frame>
  ![5 Load Latest App](/images/5-Load-Latest-App.jpg)
</Frame>

This opens the most recently generated application preview for continued validation and testing.

## Handle Build Failures

In some situations, unsupported configurations, widgets, workflows, or application events may prevent the application preview from building successfully.

When a build issue occurs, Symplr displays the build status together with the related error information.

<Frame>
  ![8 Build failed message](/images/8-Build-failed-message.jpg)
</Frame>

The error details help identify issues that may require additional troubleshooting or application updates.

### Rebuild the Application

After correcting the identified issue, click the **Build Again** button to restart the application build process.

<Frame>
  ![9 Build Again Button](/images/9-Build-Again-Button.jpg)
</Frame>

Symplr then attempts to generate a new application preview using the updated application configuration.

### Contact Support

If the issue persists after rebuilding the application, use the **Contact Support** option available within the build failure screen.

<Frame>
  ![10 Contact Support Button](/images/10-Contact-Support-Button.jpg)
</Frame>

Use this option if additional troubleshooting or support assistance is required.

## Next Steps

After validating the application preview, you can continue by downloading the generated code, running the application locally, configuring authentication and database integrations, extending workflows, and deploying the application to external environments.

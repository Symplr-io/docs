---
title: "Build & Run Applications"
---

Symplr allows you to build and preview applications directly within the platform before downloading or deploying the generated application package.

The Build & Run capability helps validate application screens, navigation flows, layouts, forms, interactions, and overall application behavior inside a browser-based preview environment.

## Before You Begin

Before building and running your application, ensure that:

- Your project has been created or opened successfully
- The application structure is available within the **Blueprint View**
- Screens, widgets, navigation flows, and workflows have been configured
- The application is ready for preview and validation

### Step 1: Open the Build & Run Menu

Navigate to the **Blueprint View** of your project.

<Frame>
  ![1 Blueprint View before build](/images/1-Blueprint-View-before-build.jpg)
</Frame>

From the top-right corner of the Blueprint View, click the **Build & Run** dropdown icon.

<Frame>
  ![2 Build & Run option](/images/2-Build-&-Run-option.jpg)
</Frame>

The Build & Run menu provides access to application preview, build generation, and runtime validation capabilities.

### Step 2: Start a New Application Build

Within the Build & Run menu, click the **Build** button under **Build & Run New App**.

<Frame>
  ![3 Build & Run button](/images/3-Build-&-Run-button.jpg)
</Frame>

Symplr begins generating a runnable preview of the application using the current application structure, configured widgets, navigation flows, and workflows.

The build duration may vary depending on the size and complexity of the application.

### Step 3: Wait for Build Completion

While the application is being prepared, Symplr displays the current build progress and generation status.

<Frame>
  ![4 Build in progress state](/images/4-Build-in-progress-state.jpg)
</Frame>

During this process, Symplr prepares the preview environment and generates the required application runtime resources.

### Step 4: Validate the Running Application

Once the build completes successfully, the generated application preview opens directly within the browser.

<Frame>
  ![6 Successful application preview running in browser](/images/6-Successful-application-preview-running-in-browser.jpg)
</Frame>

You can then interact with the generated application to validate layouts, navigation, workflows, forms, and overall application behavior.

For example, users can navigate between screens and validate application interactions directly within the preview environment.

<Frame>
  ![7 Validation example](/images/7-Validation-example.jpg)
</Frame>

The preview environment helps verify the application experience before exporting or deploying the generated application package.

## Load a Previously Generated Build

If a successful application build already exists, Symplr allows the latest generated preview to be loaded directly without rebuilding the application again.

From the **Build & Run** menu, click the **Load** button under **Load Latest App**.

<Frame>
  ![5 Load Latest App](/images/5-Load-Latest-App.jpg)
</Frame>

This loads the most recently generated application preview directly within the browser for continued validation and testing workflows.

## Handle Build Failures

In some situations, unsupported configurations, widgets, workflows, or application events may prevent the application preview from building successfully.

When a build issue occurs, Symplr displays the build status together with the related error information.

<Frame>
  ![8 Build failed message](/images/8-Build-failed-message.jpg)
</Frame>

The displayed error information helps identify issues that may require application changes or additional troubleshooting.

### Rebuild the Application

After correcting the identified issue, click the **Build Again** button to restart the application build process.

<Frame>
  ![9 Build Again Button](/images/9-Build-Again-Button.jpg)
</Frame>

Symplr then attempts to generate a new application preview using the updated application configuration.

### Contact Support

If the issue persists after rebuilding the application, use the **Contact Support** option available within the build failure screen.

<Frame>
  ![10 Contact Support Button](/images/10-Contact-Support-Button.jpg)
</Frame>

This allows the issue to be escalated for additional troubleshooting and support assistance.

## What You Can Validate During Preview

The browser-based preview environment can be used to validate:

- Screen layouts and rendering
- Navigation flows between screens
- Forms and user interactions
- Widget placement and behavior
- Responsive application layouts
- Workflow execution and application logic
- Overall application usability and experience

## Next Steps

After validating the generated application preview, you can continue with:

- Downloading the generated application code
- Running the application locally
- Configuring authentication and database integrations
- Extending the generated application workflows
- Deploying the application through external environments

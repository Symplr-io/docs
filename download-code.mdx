---
title: "Download Code"
---

Download the generated application source package from Symplr to continue development, testing, customization, and deployment workflows outside the platform.

The generated code package contains the application source files produced from your application structure, screens, workflows, widgets, and configuration settings defined within Symplr.

## Before You Begin

Before downloading the generated application code, ensure that:

- Your project has been created or opened successfully
- The application structure is finalized within the **Blueprint View**
- You have access to the **Developer Tools** menu
- The application is ready for code generation

### Step 1: Open Developer Tools

Navigate to the **Blueprint View** of your project.

From the top-right corner of the Blueprint View, click the **Developer Tools** dropdown icon.

<Frame>
  ![1 Optional Developer Tools Access Screenshot](/images/1-optional-Developer-Tools-access-screenshot.jpg)
</Frame>

The Developer Tools menu provides access to application code generation and export capabilities.

### Step 2: Configure Generation Options

Within Developer Tools, you can configure generation options before exporting the application source package.

<Frame>
  ![2 Developer Tools](/images/2-Developer-Tools.jpg)
</Frame>

Depending on your requirements, you can:

- Include generated test cases within the exported package
- **Add Security Features** — Includes security-related protections during application generation
- Connect the project to GitHub for version control workflows

These options help prepare the generated application package for testing, validation, customization, and continued development workflows.

### Step 3: Generate the Application Code Package

After configuring the required options, click the **Download** button within Developer Tools.

Symplr begins generating the application source package using the current application structure and selected generation settings.

<Frame>
  ![3 Download In Progress Confirmation](/images/3-Download-in-progress---confirmation.jpg)
</Frame>

Once the generation process starts successfully, Symplr displays a confirmation message indicating that the code package is being prepared.

### Step 4: Receive the Generated Code Package

After the generation process completes, Symplr sends an email notification to your registered email address.

The email contains a **Download code** button that allows you to download the generated application package.

<Frame>
  ![4 Email Content](/images/4-Email-content.jpg)
</Frame>

The generated package is provided as a downloadable compressed source package.

### Step 5: Download the Generated Package

From the email notification, click the **Download code** button to download the generated application package to your local machine.

<Frame>
  ![5 Downloaded Code](/images/5-downloaded-code.jpg)
</Frame>

Once downloaded, the package becomes available within your local downloads directory and can be extracted for development and testing workflows.

## What the Generated Package Can Be Used For

The generated application package can be used for:

- Running the application locally
- Extending or customizing the generated source code
- Performing additional testing and validation
- Continuing development outside Symplr
- Integrating the application into external development workflows
- Deploying the application through external environments and pipelines

## Next Steps

After downloading the generated application package, you can continue with:

- Running the application locally
- Configuring local development environments
- Extending the generated application code
- Integrating external services and APIs
- Configuring authentication and deployment workflows

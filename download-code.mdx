---
title: "Download Code"
---

Symplr allows you to generate and download the application source package so development, testing, customization, and deployment workflows can continue outside the platform.

The generated code package contains the application source files produced from your application structure, screens, workflows, widgets, and configuration settings defined within Symplr.

## Before You Begin

Before downloading the generated application code, ensure that:

- Your project has been created or opened successfully
- The application structure is finalized within the **Blueprint View**
- You have access to the **Developer Tools** menu
- The application is ready for code generation

### Step 1: Open Developer Tools

Once your application structure has been finalized within the Blueprint View, the next step is to generate the application source package for external development workflows.

To begin the export process, navigate to the **Blueprint View** of your project and open the **Developer Tools** menu from the top-right corner.

<Frame>
  ![1 Optional Developer Tools Access Screenshot](/images/1-optional-Developer-Tools-access-screenshot.jpg)
</Frame>

The Developer Tools menu provides access to application code generation and export capabilities.

### Step 2: Configure Generation Options

Before generating the application package, Developer Tools allows additional generation capabilities to be included based on your development and testing requirements.

For example, generated test cases can be included to support testing and validation workflows, while security-related protections help strengthen the generated package before continuing development outside Symplr.

<Frame>
  ![2 Developer Tools](/images/2-Developer-Tools.jpg)
</Frame>

- **Generate Test Cases** — Includes generated testing resources within the exported package
- **Add Security Features** — Includes security-related protections during application generation

These capabilities help support testing, validation, and continued development workflows outside Symplr.

### Step 3: Generate the Application Code Package

Once the required generation options have been configured, proceed with generating the application source package using the **Download** option available within Developer Tools.

Symplr then begins preparing the generated application package using the current application structure and selected generation settings.

<Frame>
  ![3 Download In Progress Confirmation](/images/3-Download-in-progress---confirmation.jpg)
</Frame>

Once the generation process starts successfully, Symplr displays a confirmation message indicating that the code package is being prepared.

### Step 4: Receive the Generated Code Package

After the generation process completes successfully, Symplr delivers the generated application package to your registered email address.

The email notification includes a **Download code** button that allows the compressed source package to be securely downloaded for continued local development workflows.

<Frame>
  ![4 Email Content](/images/4-Email-content.jpg)
</Frame>

The generated package is provided as a compressed source package for continued local development.

### Step 5: Download the Generated Package

From the email notification, click the **Download code** button to download the generated application package to your local machine.

Once downloaded, the package can be extracted locally and used to continue development, testing, customization, and deployment workflows outside Symplr.

<Frame>
  ![5 Downloaded Code](/images/5-downloaded-code.jpg)
</Frame>

## What the Generated Package Can Be Used For

Once downloaded, the generated application package can be used to continue development workflows outside Symplr, including local development, application testing, customization, and deployment preparation.

- Running the application locally
- Extending or customizing the generated source code
- Performing additional testing and validation
- Continuing development outside Symplr
- Integrating the application into external development workflows
- Deploying the application through external environments and pipelines

## Next Steps

After downloading the generated application package, you can continue with:

- Running the application locally
- Configuring local development environments
- Extending the generated application code
- Integrating external services and APIs
- Configuring authentication and deployment workflows

## Why It Matters

- Transform application ideas into working structures within minutes
- Reduce manual setup by generating screens, workflows, and layouts automatically
- Validate application concepts early before development begins
- Accelerate prototyping and experimentation using AI-assisted generation
- Move from idea to Blueprint View without writing code

## Overview

Create applications in Symplr by describing your application requirements using natural language prompts.

Symplr uses AI-assisted generation to transform application ideas into structured screens, navigation flows, and reusable layouts.

Instead of manually creating screens and workflows, you can describe the expected functionality, user experience, and application behavior directly within the AI prompt workspace.

## What You Can Do with Symplr

- Generate applications using simple natural language prompts
- Create screens, layouts, and navigation automatically
- Define workflows, user interactions, and application behaviors
- Accelerate prototyping without manual screen creation
- Continue customizing the generated application within Blueprint View

## Quick Start

Turn your application idea into a working structure in just a few steps.

You can:

- Describe your application idea
- Generate screens and navigation
- Review the generated Blueprint View
- Preview and customize generated screens

## Expected Outcome

After completing this process, you will have:

- A generated application structure based on your prompt
- Screens and navigation configured automatically
- A Blueprint View ready for customization and validation
- A generated application ready for customization, validation, and continued development

## What Gets Generated

Based on the provided prompt, Symplr can automatically generate:

- Application screens and layouts
- Navigation flows
- Forms and reusable UI components
- Workflow structures
- Initial application configurations

Generated results can be further customized within Blueprint View after creation.

## What You’ll Need

Before you get started, ensure you have:

- Access to the Symplr workspace
- A signed-in Symplr account
- Application requirements or ideas prepared for prompting
- A stable internet connection

## Example AI Prompt

The following example demonstrates a task management application prompt:

> Create a task management mobile application for teams and individual users.
>
> The application should include:
>
> - User login and registration
> - Dashboard displaying task summary and upcoming deadlines
> - Create, edit, delete, and assign tasks
> - Task status management (To Do, In Progress, Completed)
> - Task priority levels (Low, Medium, High)
> - Due date and reminder support
> - Team member list and task assignment
> - Search and filter tasks by status, priority, or assignee
> - Notifications for upcoming or overdue tasks
> - Profile and settings screen
> - Responsive mobile-friendly layouts
> - Clean modern UI with intuitive navigation

## Build & Run Your Application

After reviewing the generated application structure, you can build and run the application directly within Symplr or on a physical mobile device.

Symplr allows you to:

- Build the generated application
- Preview screens and workflows
- Validate layouts and navigation
- Run the application on a real device using Expo Go

Use the Build & Run capabilities to validate the generated application before continuing customization or code generation.

For detailed instructions, see:

- [Build & Run App](https://docs.101studio.co/build-and-run-applications)
- [Run on Device with Expo Go](https://docs.101studio.co/run-on-device-with-expo-go)

## Need Help?

If the generated result does not match expectations:

- Review and refine the prompt structure
- Add more details about workflows and screens
- Clarify workflows or navigation requirements
- Regenerate the application after updating the prompt

If issues continue, contact Symplr Support for assistance.

## Next Steps

After generating the application, continue by customizing screens, configuring workflows, applying themes, validating layouts, generating application code, or running the application through Build & Run workflows.

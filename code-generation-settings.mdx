Developer Tools allows you to configure additional capabilities before generating and downloading your application code from Symplr.

These capabilities help prepare the generated application package for testing, validation, security-related protections, and continued development outside the platform.

## Before You Begin

Before using Code Generation Settings, ensure that:

- Your project has been created or opened successfully
- The application structure is available within the project
- You have access to the **Blueprint View**
- The application is ready for code generation or export

### Step 1: Open Developer Tools

After finalizing your application within the Blueprint View, you can access additional code generation capabilities through the **Developer Tools** menu.

From the top-right corner of the Blueprint View, click the **Developer Tools** dropdown icon.

  ![Image](/images/image.png)
</Frame>

The Developer Tools menu provides access to additional capabilities related to code generation, testing, security, and code export workflows.

### Step 2: Configure Code Generation Options

The Developer Tools menu provides additional capabilities to help prepare the exported application package for testing, security-related protections, code export, and external development workflows.

Depending on your requirements, you can include generated test cases, apply additional security-related protections, download the generated code package, or connect the project to GitHub.

These capabilities support testing, validation, collaboration, and continued development workflows outside Symplr.

<Frame>
  ![Developer Tools](/images/Developer-Tools.jpg)
</Frame>

Within Developer Tools, the following capabilities are available during application code generation:

- **Generate Test Cases** — Includes generated testing resources within the exported application package
- **Add Security Features** — Includes additional security-related protections during application generation
- **Download Code** — Generates and exports the application source code package
- **Connect GitHub** — Connects the project to GitHub for version control and collaboration workflows

These options help prepare the generated source package for testing, validation, customization, and continued development workflows.

## Generate the Application Code Package

After selecting the required generation options, proceed with downloading the application code package using the **Download** button available within Developer Tools.

Symplr generates the application source package based on the selected configuration options and prepares it for delivery.

The generated code package is delivered to your registered email address, allowing you to download and continue working with the application locally.

## What the Generated Package Can Be Used For

Once downloaded, the generated application package can be used for local development, application testing, customization, and continued development workflows outside Symplr.

- Running the application locally
- Extending or customizing the generated code
- Performing additional testing and validation
- Continuing development outside Symplr
- Integrating the application into external development workflows

## Next Steps

After configuring code generation settings and downloading the generated package, you can continue with:

- Downloading the generated application code
- Running the application locally
- Testing the generated application package
- Extending the application with additional workflows and integrations
- Configuring authentication and external services
